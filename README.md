# 📸 Screenshot Share App (MVP)

## 🧭 概要

ゲーマー向けに設計された**スクリーンショット共有アプリ**です。  
第一弾 MVP では、以下の 3 つを軸に構成されています：

- **保存数制限（無料：20 枚）＋課金解除（有料：無制限）**
- **共有アルバム機能によるコラボ体験**
- **Supabase ＋ React による軽量・高速な構成**

---

## 🧱 技術スタック

| 項目           | 採用技術・理由                                                 |
| -------------- | -------------------------------------------------------------- |
| フロントエンド | React / Next.js（高速開発・状態管理しやすい）                  |
| バックエンド   | Supabase（Auth, DB, Storage, RLS）                             |
| 課金処理       | Stripe（サブスクリプション型）                                 |
| 状態管理       | Zustand or React Context（グローバルに課金状態・保存数を管理） |
| デプロイ       | Vercel（CI/CD と相性が良い）                                   |

---

## 💸 収益化設計（MVP）

| プラン       | 内容                                                       |
| ------------ | ---------------------------------------------------------- |
| 無料ユーザー | スクショ保存上限：20 枚、広告表示あり                      |
| 有料ユーザー | 保存数無制限、広告非表示、共有アルバム制限解除（将来的に） |

- 保存数制限は UX を損なわずに課金動機を生む設計
- Stripe 連携によりサブスク管理を自動化
- 将来的に広告 SDK 導入も検討（MVP では未実装）

## 🔐 セキュリティ設計（RLS）

Supabase の Row Level Security（RLS）を活用し、以下のようにアクセス制御を実装：

- `screenshots`: 自分のスクショのみ閲覧・操作可能
- `albums`: 作成者 or メンバーのみ閲覧可能
- `album_members`: 自分の所属情報のみ取得可能
- `users`, `subscriptions`: 自分の情報のみ取得可能

詳細は [`docs/logic/rls-policy.md`](./docs/logic/rls-policy.md) を参照。
